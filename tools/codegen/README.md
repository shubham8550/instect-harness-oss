# `tools/codegen/`

JSON Schema → **Pydantic v2 + Zod + Rust** codegen for the `insect.eval.v1`
family of schemas.

## Why this exists

`harness-oss` is polyglot (Rust daemon, Python SDK, TypeScript SDK). To keep
those three surfaces from drifting we author types **once** as JSON Schema in
`schemas/` and project them into each language with a single pipeline.

## Layout

```
tools/codegen/
├── codegen.py            # entrypoint; orchestrates the three emitters
├── codegen_python.py     # Pydantic v2 via datamodel-code-generator
├── codegen_typescript.py # Zod + inferred TS via json-schema-to-zod (Node CLI)
├── codegen_rust.py       # Rust types via typify CLI
├── requirements.txt      # Python deps (pinned)
└── README.md             # this file
```

Outputs land in:

* `python/packages/instect_core/src/instect_core/_generated/`
* `typescript/packages/core/src/_generated/`
* `crates/insect-core/src/_generated/`

Every emitted file carries an SPDX header and an `@generated` marker. Hand
edits are rejected by the drift-check (`tools/drift_check.sh`).

## Toolchain choice notes

* **Python** — `datamodel-code-generator` produces Pydantic v2 models with
  good support for `$ref`, `$defs`, and discriminated unions. The volatile
  `# generated by datamodel-codegen` and `# timestamp:` lines are scrubbed
  post-generation so output is byte-stable across runs.
* **TypeScript** — `json-schema-to-zod@2` (Node CLI) gives Zod schemas plus
  inferred TS types from a single source, matching the contract from
  `plans/P5_foundation_shared.md §4.2`.
* **Rust** — `typify` (Oxide Computer) over `quicktype` and `schemars`:
  * `typify` handles JSON Schema 2020-12 idiomatically and emits structs that
    survive `clippy -D warnings`.
  * `schemars` runs the *other* direction (Rust → Schema) — useful only for
    round-trip drift checks, not for first-pass generation.
  * `quicktype` is TS-first, weaker on 2020-12, and emits noisy `#[allow]`
    attributes.

## Adding a new schema

1. Drop a `*.json` file under `schemas/<family>/`. Use JSON Schema 2020-12.
   Give it a stable `$id` like
   `https://schemas.instect.dev/insect.<concept>.v<major>.json`.
2. Run `just codegen` from the repo root.
3. `git add -p` the regenerated files under each language's `_generated/` dir.
4. Open a PR. CI runs `bash tools/drift_check.sh` to verify the regen.

When the schemas dir is empty (the W2-C agent hasn't landed anything yet),
the pipeline emits empty-but-valid barrel files and exits cleanly.

## Local regeneration

```sh
just codegen
# or, directly:
uv run python tools/codegen/codegen.py \
  --mode=generate \
  --schemas-dir=schemas \
  --out-py=python/packages/instect_core/src/instect_core/_generated \
  --out-ts=typescript/packages/core/src/_generated \
  --out-rust=crates/insect-core/src/_generated
```

## Drift-check

```sh
bash tools/drift_check.sh
# or via the justfile alias:
just drift-check
```

The drift-check writes outputs to a tempdir and diffs them against the
committed `_generated/` files. Any difference fails with a hint to run
`just codegen`. Wired into pre-commit and the `codegen-drift.yml` CI workflow.

## Determinism guarantees

* Inputs are processed in lexicographic order.
* Toolchain-volatile lines (timestamps, absolute paths, generator banners)
  are stripped from emitter output before write.
* All inputs and outputs use `\n` line endings (enforced by the
  pre-commit `mixed-line-ending` hook).
* SPDX + `@generated` headers are emitted from our own templates, not from
  the underlying generator.

Byte-stability is best-effort against upstream-generator quirks; the
drift-check is the authoritative gate.
